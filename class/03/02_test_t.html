문제18) xhtml은 비표준과 표준 어디에 속할까?

xml(1.0) + html == xhtml1, 1.0 || 1은 모두 표준 버전에 속함

문제19) 비표준은 몇 버전이고, 표준은 몇 버전인지 작성하기

표준은 1.0 || 1
비표준은 4.0 || 4
2개의 문서를 합쳐서 5.0 이 아니라 1버전과 4버전을 호환하는 문서가 존재함

문제20) 표준과 비표준을 호환하는 문서는?

transitional.dtd, (모든 언어를 호환할 수 있음)
단 charset="utf-8"로 되어있어야함

utf-8로 모든 언어를 작성할 수 있어야하고, transitional.dtd로 해당하는 언어를 호환할 수 있어야 한다

utf-8로 모든 언어를 작성할 수 있어도, strict.dtd라면... 소문자 표현만 가능하기 때문에 언어지원에 문제가 발생함

문제21) 표준언어는 어떤 언어를 주로 사용할까?

영문 소문자만 사용이 가능함

문제22) p태그안에 div태그를 담을수 없는 이유는?

div태그는 폴더처럼 사용
p태그는 파일처럼 사용

1)가장 작은 박스 p태그에 div태그 가장 큰 박스를 넣을 수 없음

2)파일안에 폴더를 넣을 수 없음/폴더 안에 파일을 넣을 수 있음

문제23) 압축할때 문자열이 깨졌다면 이유는? 그리고 해결방법은?

utf-8로 문자열이 무겁기때문에 ANSI방식으로 문자열이 바뀌게 된다
해결방법으로는 새로 저장할 때 utf-8로 저장하면 해결이 가능함

단, 처음부터 바꿔야가능함. 중간 어느정도 코딩을 한 다음 바꾸게 되면
문자열은 어떻게 표기할 것인지 알림 창이 뜨게 됨! 이 때 문자열이 전부
깨져버릴 수 있기때문에 주의해야 함
(초기에 utf-8로 되어있는지 반드시 확인하기)

문제25) 인라인요소를 블록요소에 담아서 사용하는 것을 권장하는 이유는?

대부분의 인라인 요소들은 중요한 속성을 가지고있음, img태그의 경우 src,alt등의 중요 속성을 가지고있음
:: 해당하는 태그들을 보호하기 위하여 블록속성의 태그에 담아서 표기한다.

매우중요)인라인요소는 스스로 너비와 높이를 만들지못함, 담고있는 블록요소를 이용하여 너비와 높이를 만들 수 있음
