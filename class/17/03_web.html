웹 표준, 웹 접근성의 규칙

  1. 텍스트가 아닌 컨텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.
  2. 멀티미디어 컨텐츠는 자막, 대본 또는 수화 제공을 해야한다.
  3. 컨텐츠는 색에 관계 없이 인식될 수 있어야 한다.(색맹,색약)
  4. 텍스트 컨텐츠의 명도 대비(지나치게 차이가 나지 않도록 설정한다)
  5. 배경음 사용금지(사용자 요구에 의한 재생)
  6. 컨텐츠 간의 구분이 명확해야한다. 이웃한 컨텐츠는 구별할 수 있어야한다.
    (hr구분선, 배경색상,배경이미지 등으로 어느 영역인지 구분할 수 있어야한다.)
  7. 키보드 사용 보장, 모든 기능은 키보드만으로 사용할 수 있어야한다.
  8. 초점이동, 키보드에 의한 초점은 논리적으로 이동해야하고, 시각적으로 구별할 수 있어야한다.(tab키 사용)
  9. 사용자 입력 ctrl 조작이 가능하도록 제작되어야한다.(슬라이딩윈도우, 멈춤과 재생 필수)
  10. 응답시간 조절, 시간제한이 있는 컨텐츠는 응답 시간을 조절할 수 있어야한다.
  11. 정지 기능 제공, 자동으로 변경되는 컨텐츠는 움직임을 제어할 수 있어야한다.
  12. 제목제공 (title,headline), 페이지(title), 프레임, 컨텐츠(headline)에는 적절한 제목을 제공해야 한다.
  13. 기본 언어 표시, 주로 어떤 언어를 사용하는지 표기해야한다.(lang="ko" xml:lang=kor)
  14. 사용자 요구에 따른 실행 (정지버튼, 컨트롤 사용등을 보장한다.)
  15. 컨텐츠의 선형 구조 - 논리적인 순서로 왼쪽에서 오른쪽으로 움직일 수 있어야한다.
  16. 표의 구성, 표는 이해하기 편하게 구성되어야 한다.(회사의 조직도, 페이지의 목차)
  17. 오류정정, 입력한 오류를 정정할 수 있는 방법을 제공해야한다
    (아이디를 8자 까지만 입력한다면 8자까지만 입력할 수 있도록 알림창 제공)
  18. 마크업 오류방지, 마크업 언어의 요소는 열고 닫음, 중첩 관계, 속성 선언에 오류가 없어야 한다 (validator, kwah(접근성 100점 도구), id 중첩, 공간 중첩 margin, padding 겹치는 문제...)
  19. 멀티미디어 컨텐츠는 초당 3~50회 이상 깜빡임이 없어야한다.
  20. 웹 접근성을 준수한다.
