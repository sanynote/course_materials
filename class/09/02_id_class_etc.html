lt에 대한 이해

+ 참고 사이트: 삼성엔지니어링(구형 브라우저 & 반응형 페이지 제작)

+ 구형 브라우저에 맞추는 작업: lt와 gt를 사용함
+ 구형 브라우저는 "익스플로러"만 존재함!
+ if lt IE 7 : 만약 인스터 익스플로러 7버전 "이전 (lt)" 버전이라면....
+ IE 7 : 익스플로러 7버전

gt에 대한 이해

+ if gt IE 8: 만약에 인터넷 익스플로러 8버전 이후 버전이라면...
+ IE 8: 익스플로러 8버전

현재 버전, 이전, 이후 버전

+IE 11: 현재 버전을 작성할때
+if lt IE 10 : 10버전을 제외한 이전 버전들:9,8,7,6,5
+if gt IE 8: 8버전을 제외한 이후 버전들:9,10,11
+ 삼성엔지니어링 사이트에서 확인해보면 현재 버전들을 포함하지 않아 따로 작성한것을 확인할 수 있음

+ if lte IE 10: 현재 버전 10을 "포함"한 이전 버전들:10,9,8,7...
+ if gte IE 8: 현재 버전 8을 "포함"한 이후 버전들:8,9,10,11

매우중요) lte,gte는 현재 버전을 포함하지만 "인식률이 낮아" 삼성 엔지니어링 사이트처럼 따로 구분하여 사용하는 경우가 많다

id속성 & class속성

id 속성이란?

네이밍을 만들때 중요사항
id의 경우, 코드생략으로 태그를 닫았을때 id만 보고 무엇을 담고있는지
알 수 있어야 함

1) 한 페이지에서 같은 이름을 2번 사용하기 어려움
2) 부모나 영역이 되는 공간에 사용, #header p
3) 예약된 용어들을 대신할 때 사용
(header, nav, gnb, container, visual, section(headline, h2~h6), aside, footer)

4) 고정된 기능명으로 영역을 다룰때 (scroll_menu, click_area)
5) style과 script 영역에서는 id값을 한 번만 다루기때문에 가장 빠르게 인식함
6) body에서는 tag부터 작성되어야 id를 작성할 수 있기때문에 tag 부터 읽어들임
7) style에서 #으로 호출가능하고, jQuery에서도 #으로 호출가능함

class 속성이란?

1) id는 유일하게 한 번만 이름을 사용하기때문에 대량작업시 제약이 많음
2) 그룹화, 개별작업,대량작업 등 사용자가 원하는대로 만들기위하여 class를 많이 사용함
3) 중요) class는 같은 이름을 여러번 사용할 수 있음 4) 매우 중요) id는 주로 예약된 용어나 인식이 가능한 이름을 사용하지만
class는 사용자가 압축해서 이름을 만들거나 기능명 등으로 사용할 수 있음(button(btn), click(clk))

확실하게 알아두기) id="header" class="header", 속성이 다르기때문에 같은 이름 사용가능함
1) id="header", id="header", id 속성이 2번 같은 이름을 사용했기때문에 사용 불가능
2) class="header", class="header", class속성은 그룹화가 가능하기때문에 사용가능함

id는 주로 부모나 큰 영역에 지정하고, class는 자녀나 기능이 적용되어야하는 부분에 주로 많이 사용함

인식률

브라우저에서 인식할 수 있는 확률 (tag, id속성, class속성)

브라우저가 활성화 된 순간 읽는 순서는? (속도)

body가 열린부분부터, body가 닫힌 부분까지 위에서 아래로 순서대로 읽어들임
속도가 매우 빠르기때문에 동시에 읽는것처럼 느껴짐

인식률은 style, script, php, "body를 제외"하고 나머지 모든 영역에서는 id속성이 가장 빠르다.
작성된 이름이 하나밖에 없기때문에 인식률이 떨어질 수 없음

중요) body는 tag부터 읽음, id와 class는 tag가 없다면 작성이 불가능함

style과 script, php영역에서 class나 tag를 호출할때, 같은 이름이 많기때문에 인식률이 떨어질 수 있음
인식률이 떨어지면, style효과 적용 불가능, script 기능 적용 불가능, php db적용 불가능

정리)body는 tag, body를 제외한 나머지 영억은 id속성 제일 빠르게 읽음

인식률을 높이는 방법)
.text, 같은 이름의 class를 여러군데 설정할 수 있기때문에 어느 영역의 class인지 알기 어려움
+ #header .text , 어느 영역의 class인지 정확하게 작성해준다.
+ #header p , 어느 영역의 tag인지 정확하게 작성해준다.

속도가 느려지는 경우

1) body가 시작되고, 닫히는 부분까지 위에서 아래로 읽음

2) 태그만 있을경우, id와 class가 없기때문에 태그만 읽을 수 있음

3) p id="text" , 태그도 읽고, id도 읽어야하기때문에 느려질 수 있음

4) p id="text" class="desc" , 태그, id, class 모두 읽어야 하기 때문에 가장 느려질 수 있음

5) 기본적으로 느려진 페이지는 다른 것으로 속도를 보완해도 느려질 수 있기때문에 주의해야한다.

매우 중요) 작업중인 경우에는 많이 사용할 수 밖에 없음
최종적으로 확인할때 사용하지 않는 id나 class, tag가 있다면 지워주는 "경량화" 작업을한다.
