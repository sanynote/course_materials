반응형 VS 절대형 VS 적응형

반응형, 절대형, 적응형 모두 통합해서 반응형 페이지라고 읽음

핵심: 반응형,절대형,적응형 모두 x축으로 scroll이 나타나지 않도록 설계해야 함

반응형
정리: 작은 움직임에도 반응하는 페이지가 반응형 페이지
주로 % 단위로 작업함, 작은 단위의 값들도 따라다닐 수 있도록 만듦
단점: %위주로 작업하기때문에 해당하는 영역이나 객체, 이미지가 쉽게 깨질 수 있음 (비율) : 목적이 상실됨, 제품을 표현하는 영역에서 깨지면 안되는 이미지나 설명글들이 제대로 확인이 불가능함
장점: 모든 해상도를 따라서 움직이기때문에 해상도가 변화될 때 부드럽다
%를 잘못사용하는 경우, animation 풍선 예제 부모가 없거나 px로 고정된 영역이 없을 경우, 더 큰 부모를 따라다니는 문제 발생
매우중요) %를 사용하기 전에, 담아주는 대상에 px이 있는지 반드시 확인하기
절대형(적응형)
정리: 정확한 크기를 고정시켜두고 사용하는 페이지
반응형 페이지는 주로 % 위주로 작업함
절대형 페이지는 주로 px로 작업함
절대형 페이지: 반응형 페이지보다 실제로 더 많이 사용되는 페이지 제품을 판매하는 사이트의 경우, 이미지가 늘어나거나 축소되면서 원본의 형태가 깨질 수 있기때문에 절대형 작업을 많이 한다
장점: * 팀 작업시 매우 빠른 작업이 가능하다 이미지나 어떤 대상의 값이 쉽게 변형되지 않음
단점: 움직임 없기때문에 브라우저를 줄이거나 늘릴때 변화되는 요소가 없음 매우중요) 모바일 기기에서는 브라우저를 줄일 수 없기때문에 해당하는 해상도에 맞추는 작업을 많이 함
적응형
정리: 반응형과 절대형을 섞어서 만드는 페이지 (적절한 반응형 and 적절한 절대형)
+headerWrap (전체사용영역), header(실제사용영역) + container, section id="bestArea" (100%), best(실제사용영역) + footerWrap (전체사용영역, 100%), footer(실제사용영역)
중요) 실제로 사용하는 영역은 주로 절대형 작업으로 제작함 (상품이나 이미지의 비율을 유지해야 함)
중요) 바깥쪽에 전체 사용하는 % 영역 만들고, 실제로 사용하는 px 영역 만들어서 움직일 수 있도록 표현하기 (참고, 알툴즈페이지, wide 로 표현되어있지만 실제로는 안에 고정된 영역이 있어서 깨지지않음)
