body영역에 script를 작성하는 이유는?

브라우저에서 읽어주는 순서 필수로 확인하기

1) body영역
2) style영역
3) javascript 영역
4) jQuery는 javascript 없으면 사용하지 못 함, 마지막에 읽는 영역!

body영역은 브라우저 활성화시 브라우저가 준비가 되어야 하기 때문에 가장 빠르게 읽는 영역
* script영역을 body영역에 작성하면 어느정도 속도를 보완할 수 있음!
* body영역은 가장 상단이나 하단에 작성함!

1) script 는 style처럼 분해하여 사용이 가능함
2) body에서 소스줄만 연결해서 사용하는 경우도 많음
3) body영역에서는 여러가지 소스들이 연결되어 꼬여 있을수 있기 때문에 중간에 작성하는 것은 굉장히 위험함!

기본 요소 선택자
tag, id, class 접근하기
모든 "공간" 은 0번 부터 시작함
모든 "숫자" 는 1번 부터 시작함
영역 순서_활용하기
1) body영역부터 읽음
2) style영역 읽음
3) script 영역 (javascript, jQuery) 읽음
body 1차 영역, style 2차 영역, script 3차 4차 영역 기능이나 효과를 적용하면 script 영역이 가장 마지막이기 때문에 최신의 값을 담당하는 경우가 많음!
* script의 느려지는 효과를 html5의 동적인 요소와 섞어서 사용할 수 있다면 안전하고 빠르게 만들 수 있음
