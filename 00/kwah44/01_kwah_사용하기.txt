

1) 설치하기
	+ 설치가 되어있는 컴퓨터에서는 "복구하기" 다음,다음 완료

2) 실행파일 확인하기
	+ kwah44 폴더 확인
	+ kwah44_setup.msi 설치하기

3) 설치완료, 닫기

4) 바탕화면에 사람아이콘 확인 > 실행 > 업데이트 불가능 (아니오)

5) framework 오류 활성화되면 "끝내기" 클릭 || script 오류들 활성화되면\
"아니오" || no || 끝내기

+++++++++++++ [설정하기] ++++++++++++

1) "프로젝트를 저장"하기위한 목적으로 폴더 생성
2) 가장 상단의 메뉴들 중에서 "설정"부분 클릭
3) 점검설정 열기 클릭

4) 환경설정 창에서 가장 상단의 "디렉토리 설정" 항목 부분 확인
5) 경로와 오른쪽 찾기버튼
6) 찾기버튼 클릭
7) 내 PC (내 컴퓨터) > c드라이브 > 새폴더만들기 버튼 클릭 > kwah라는 이름의 폴더 만들기 > 확인 > 저장 > 확인

* 해당하는 폴더가 없기때문에 점검 도구가 저장할 위치를 찾을 수 없음
* 에러가 발생하는 항목으로 폴더가 있어야 안전하게 점검이 끝남

++++++++++++ [매우중요 , 업로드] ++++++++++++

1) 포트폴리오 || 개인작업 || 프로젝트 , filezilla (알드라이브) 를 통해서 업로드 확인

ex) audi 페이지 업로드

2) 개인아이디명 .dothome.co.kr/audi(폴더명)/index.html

* 브라우저에서 활성화할때는 index를 제외할 수 있지만 검사를 통해서 
해당하는 파일을 확인해야하기때문에 파일명 제외 불가능

[매우중요]
	+ 폴더에 검사할 파일 하나만 있어야 함 (검사한 html파일이 하나만 있도록 한다.)
	+ 폴더 안에 .html파일들 모두를 점검하기때문에 느려질 수 있음

++++++++++++++++ [kwah 프로그램에서 확인하기] +++++++++++++++

1) 상단의 메뉴들 중에서 "파일" 선택

	파일>새로만들기>프로젝트
		* 프로젝트 이름 : 한글사용 가능하지만 안전을 위해서 영어소문자
		* audi, kakao, naver

2) 하단의 점검 대상 부분 확인하기
	
	+URL평가 선택(필수)
	+파일평가 부분은 평가 도중 프로그램이 멈추거나 제대로 평가되지 않을 수 있음
	(비디오,오디오,이미지,압축파일 즉 모든파일들을)모두 확인하기때문에 매우 느려지거나 쉽게 멈출 수 있음

	* 컴퓨터를 재부팅해야할수도있음

3) 중간의 "평가할 홈페이지" 영역 확인
	중요) url을 검사하기위하여 index.html 파일까지 모두 작성되어야 함
	
	url: sunnyjuice94.dothome.co.kr/canada/index.html

4) 브라우저 통해 경로 열리는지 확인하고 해당 주고 복사해두기
활성화 되었다면 url 항목에 해당하는 주소 붙여넣으면 됨

5) 프로젝트 이름 비어있지 않은지 확인, url에 주소줄 붙여넣기 + 프로젝트 이름 작성

6) 상단의 프로젝트 이름 옆에 큰 "시작" 버튼 클릭>확인>잠시대기

7) 오류발생시
	왼쪽 경로 부분에 +형태의 마크가 활성화 됨
	+ 버튼 누르고 확인해보면 validator처럼 오류나는 행과 열이 있는것을 확인할 수 있음

8) 오류 해당부분 수정하고 반드시 업로드하기
	: 1번부터 반복작업

9) 오른쪽 상단의 보고서 보기 버튼 클릭, 100점 맞은 평가도구 캡쳐하고, ppt에 소개하는 부분 만들기 (필수)
