+++++++++++++++++++++++ database 백업 +++++++++++++++++++++

1) xampp 실행 > MySQL 줄에 Admin 버튼 클릭

2) [주의사항]
	1) xampp 를 지우면 만들어진 database나 table이 모두 지워짐!
		: xampp 지우기 전에 백업받아야 함!

	2) phpMyAdmin 페이지에 암호걸려 있으면 접속할때 분홍색(빨간색) 
	페이지가 출력되고 접속이 불가능함!

3) [백업하고 싶은 db 선택]
	
	1) 왼쪽 메뉴에서 백업하기 위한 db 선택하기
		ex) db1 (하나씩 백업하기!)

	2) phpMyAdmin 첫 화면에서 오른쪽 상단의 데이터베이스 탭 클릭해서 db 백업가능!
		ex) db1 (하나씩 백업하기!)

	3) 백업하고 싶은 db 클릭하기

[매우중요]
	내보내고 싶은 테이블 선택하기
	하나만 선택이 가능, 다중 선택도 가능함! (체크 박스 활용하기)

4) 상단의 내보내기 탭 클릭
5) 형식들 중에서 원하는 확장자로 내보낼 수 있음, "sql" 선택하고 오른쪽 하단의 "실행" 버튼 클릭

+++++++++++++++++++++++ database 가져오기 +++++++++++++++++++++

[매우중요]
	같은 이름의 db명으로 사용 불가능!
	db1을 사용하고 싶으나 "db1"은 이미 이름이 있으므로  "dbs1이나 다른 이름" 으로 만들어서 가져와야 함!

1) 왼쪽 상단의 phpMyAdmin 로고 클릭 후.. 첫 화면으로 이동하기
2) 다른이름으로 가져올 수 있는 DB 명이 필요함! 

3) 오른쪽 상단의 "데이터베이스" 탭 클릭
3) 아래 "새 데이터 베이스 만들기" 부분 확인 

	데이터 베이스 명 : dbs1
	데이터 정렬 방식 : utf8_general_ci 선택 (거의 하단에 있음)
		(mysql ,	 default charset=utf8) 

4) "만들기" 버튼 클릭

5) 테이블은 만들지 않아도 상관없음!
	(백업 받아 놓은 테이블을 가져오기 위해 작업)

6) 오른쪽 상단의 "가져오기" 탭 클릭
	파일 선택 버튼 클릭 , 가져올 자료 선택

	+ 파일 문자셋은 utf-8로 되어 있는지 확인
	+ 형식 부분은 sql로 되어 있는지 확인

	하단의 "실행" 버튼 클릭


[매우중요]
	
	1) 백업 받은 db를 사용할 때는 db 명을 확인하고 작업하기!
		원본은 db1, 백업은 dbs1 이라면 작업시 다른 db에 작업하는 문제가 발생함!
	
	2) 백업시 db 전체를 backup 할 경우, 다른 db에서 담아주지 못하기 때문에
		가져올 수 없음!

		: table을 선택하여 backup 해야 다른 db에서 가져다 사용할 수 있음!
