유효성 검사 : required="required"
  1. 필수 정보 입력공간이 빈칸인지 아닌지 확인하는 작업
  2. 문법 맞는지 확인하기 (숫자, 문자, 한글, 영문인지 체크(대문자, 소문자))
  3. 문법에 맞지 않을 경우, 알림창이나 경고창을 활성화 시킴!
  4. input에서 제공하는 그외의 작업을 필요로 할 경우, "자바스크립트"나 "제이쿼리" 를 이용함!
  5. 문자열의 길이나, 대소문자 구분, 다른나라 언어구분 등은 script 작업을 필요로 함!
  6. input의 number의 경우, 뒤에 ▲ 형태가 있음, 기본 제공 하는 값 외에 다른 것을 사용할 경우 증감문을 이용한 script 필수

예시문 힌트주기 : placeholder="예시문"

value는 인식이 되는 값으로 script에서 비교대상의 속성으로 설정이 가능함! (인식이 되기 때문에 비교대상으로 만들 수 있음)

placeholder는 가상의 값으로 클릭하고 입력받으면 사라지기 때문에 비교의 값으로 만들 수 없음! (화면에 보이는 용도로만 사용)

[매우중요] : required, placeholder 모두 속성이 다르기 때문에 사용자의 목적에 맞춰서 함께 사용이 가능함!
필수정보이면서 , 예시문까지 만들때 사용가능!

자동완성 기능 사용하기 : autocomplete( 기본상태 on, off)
  1. 문제 풀기 (응용문제)
  2. 현재 ol 태그가 끝난 부분 아래에 ul li 태그 만들기
  3. 각각의 li 태그들 마다 id, phone, email 만들기, type맞추기
  4. phone 부분에 해당하는 input에 유효성 검사와 예시문 작성하기
  5. * id 중복되지 않도록 주의하기
  6. 필수) 만들어진 공간에 정확한 정보를 기입하고, 전송해 보기
"전송 기능을 활성화" 한 후 확인하기 (전송을 했을때만 확인가능)
: 전송이 이루어지면 정보가 브라우저에 쌓이기 때문에 기본적으로 자동완성 기능이 가능하다!
: 브라우저는 메모리를 가지고 있기 때문에 잘 못 인식되는 값이 저장되면 계속해서 잘못 저장되는 문제가 발생함
: 브라우저를 활성화 하는 순간 (ctrl + b), 메모리가 쌓이게 됨, 잘 못 된 정보로 출력되는 경우, 쿠키나 캐시 삭제하기
tip) required는 필수 정보 입력공간으로 , 비워진 채로 넘길 수가 없음
빈칸인 채로 넘길수 없기 때문에 항상 기록이 남아있는 공간!
* required로 저장되어 있는 공간은 autocomplete="off"를 사용하여 막아두는 것이 보안에 안전하다!
required (필수정보입력) , placeholder="예시문", autocomplete="off"(자동완성 끄기)

placeholder VS value

placeholder는 화면에서 인식하는 값이 아니라 가상의 값
마우스 클릭하면 블록으로 잡히 않고, 입력되는 순간 값이 사라짐!
실존하는 값이 아니기 때문에 계산식 페이지를 만들기 어려움

value 값은 매우중요한 속성
select 태그, input 태그에서 활용도가 매우 높고, 클릭하면 블록으로 잡힘
인식되는 값이기 때문에 계산식 페이지 만들때 매우 중요한 속성으로 사용됨!

* readonly를 설정하여 지워지지 않도록 표현해야 null 값이 표현되지 않는다!

select 태그 VS datatlis 태그

  1. 공통으로 option과 value 속성을 사용함!
  2. select 태그의 장점: 여러번 재선택 가능함!
  3. datalist는 input 공간에 한번 인식이 되면 재선택이 어려움!
  4. select태그는 화면 어디에든 만들 수 있는 태그가 아니라 필요한 위치에서 만들어야 함!
  5. datalist는 화면 어디에 만들어도 input 공간과 datalist의 id값만 있다면 얼마든지 호출이 가능함!
  6. datalist를 호출하기 위해서는 반드시 id값이 작성되어야 한다!
  7. select 태그는 사용자가 자주 재선택 되어야 항목에서 사용하는 경우가 많고,
    datalist는 한번 선택하고 변경하지 않아야 할때 사용한다!
  8. select 태그를 더욱 많이 사용하는 이유는?
    과목선택이나, lang 선택, 여행지 선택등 사용자가 얼마든지 변경해야 하는 경우가 많기 때문에
    현장에서 || 회원가입이나 선택에 관련된 form을 작성할때는 주로 select 태그를 활용한다!
  9. datalist의 option태그와 select태그의 option 태그는 동일한 태그로 인식이 불가능함
    datalist와 select 태그 모두 value 값으로 인식함!
