문제01) h1은 어떤 용도로 사용하는 태그인지 작성하기

1)h1은 중요도가 높기 때문에 페이지 당 한 번만 사용함
2)로고나 상징적인 요소들을 담아줄 때 사용함
3)목차 출력시 가장 큰 제목에 해당하는 태그
4)headline 태그는 h1~h6까지 중요도에 따라서 목차를 만들 수 있음

tip)목차생성을 필수로 하지 않는다면
headline태그를 사용하지 않아도 상관없음
로고의 경우 div id="logo"로 사용하는 경우도 많음

p태그는 img를 담을 수 있지만, logo의 경우 가지고 있는 상징적인
부분이 크기 때문에 div 태그를 사용한다.

문제02) meta태그란?

meta태그는 주로 head영역에 작성하고, 문서의 사용용도를 표기함

네이버에서 확인(검사창 띄우기) / meta태그 중에 width="1190"
1190px까지는 보호하여 옆으로 나열된 객체들이 떨어지지 않도록 만듦
나머지 해상도는 객체가 아래로 떨어지거나 옆으로 스크롤이 만들어짐
: 웹은 옆으로 스크롤이 늘어나는 것을 매우 불편해하기때문에 해상도 마다
맞추는 작업을 하게 됨
: "반응형 작업" (스마트폰 세로~일반pc 버전의 너비에 맞추는 작업)

문제03) body영역을 대신하는 영역명은? (해당하는 영역이 필요한 이유는?)

wrap, wrapper,기업명_wrap,appleWrap

body는 모든 영역을 담당해야 하기 때문에 사이즈 작업은 하지 않는다

wrap은 body의 일부영역을 사용할 것인지, 전체를 다 사용할 것인지 결정해주는
중요 공간(header,container,footer 등을 담을 수 있음)

문제04) body를 대상으로 style에서 작성하기 어려운 작업은? (이유는?)

body를 대상으로 "사이즈 작업(너비와 높이)"은 절대 하지 않는다.
전체 영역을 다 사용해야 여러가지 작업을 할 수 있음

문제05) 구조영역, 표현영역, 기능영역은 각각 어떤 영역들인지 작성하기

+표현영역:style영역(구조를 확인하면서 꾸며주는 작업을 하는 영역)
+기능영역:javascript,jQuery
+jQuery == javascript + query (합성어)
javascript가 없다면 jQuery를 사용할 수 없음

문제08) 사이즈 작업을 하려고 한다. 800*300은 무슨뜻인지 작성하기

문제09) 사이즈 작업이 어려운 요소들은?

"블록요소" (박스)는 처음부터 끝까지 영역을 만들기 때문에 "사이즈 작업"이 가능함
"인라인요소"는 사이즈에 상관없이 글자크기만큼만 영역을 차지함
인라인요소는 너비와 높이 작업이 불가능함! (span,strong,b,em,i,img(src,alt))

문제20) div태그를 만들면 기본적으로 작성해야 하는 태그와 이유 작성하기

div태그는 일반적으로 폴더처럼 사용, 바탕화면에서 폴더 만들면 제목이 없는 "새폴더"가
만들어짐, 현재 짝을 맞추는 태그를 물어보고 있기 때문에 제목에 해당하는 headline 태그 사용

영역을 만들때 사용하는 속성은?이라고 물어봤다면, id속성으로 해당하는 영역명을 작성함
id="wrap", id="header", id="container", id="footer"

tip) 태그 (tag들 물어보는 경우), 속성(id,class,alt,src...), 요소(인라인요소,블록요소) 구분하기
